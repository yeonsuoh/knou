# 3강. 멀티미디어 데이터(1) - 텍스트

## 3.1 텍스트

### 1. 텍스트의 개요
* 자연 언어나 인공 언어, 문자 배열 형태의 데이터

### 2. 텍스트의 표현

**(1) 문자 집합과 인코딩**
* ASCII
  * 7비트 인코딩 기호 체계
* EBCDIC
* 서유럽 문자집합
* 한글코드
* 유니코드
  * UTF-8, UTF-16

**(2)마크업 텍스트**
* 문서의 서식 작성을 지정하거나 구조적인 경계를 밝히는 특수한 문자 삽입

**(3)구조적 텍스트**
* 문서의 논리적인 구조와 설계의 구조를 표현하는 데이터 구조
* 트리 형태와 같음
  * ODA(Office Document Architecture): 문서표현에서 논리적 구조와 설계 구성 요소들의 구성과 관계를 명시하는 국제 표준

**(4)하이퍼텍스트**
* 비선형으로 그래프와 같은 구조를 가지며, 노드(node)는 텍스트의 작은 덩어리로서 링크로 연결됨
* 연결된 링크를 따라 점프하여 원하는 정보를 검색
* 하이퍼텍스트나 하이퍼미디어의 구조를 형성하는 핵심적인 두 가지 요소 : 노드, 링크

### 3. 텍스트 파일 형식

**(1) TXT(Text)**
* 기본적인 텍스트 편집기

**(2) DOC(Document)**
* MS 워드

**(3) HWP(Hangul Word Processor)**
* 한글 워드 프로세서

**(4) RTF(Rich Text Format)**
* 마이크로소트사 중심 표준화
* IBM PC나 매킨토시 - 서로 다른 기종이나 운영체제에서 동작하는 프로그램 사이 형식화된 텍스트 문서 교환 가능

**(5) HTML (Hyper Text Markup Language)**
* 인터넷 웹 문서 표준
* 호환성 가장 높음

**(6) XML(eXtensible Markup Language)**
* 웹상에서 구조화된 문서를 전송 가능하도록 설계된 표준화된 텍스트 형식
* 기존 HTML 확장 및 보완

**(7) XHTML**
* HTML에 XML 적용

**(8) PDF(Portable Document Format)**
* 어도비사 개발 전자문서 형식

**(9) Tex/LaTeX**
* 수학, 자연과학, 공학 등에서 사용되는 기호나 수식이 포함된 문서를 잘 정형하고 조판할 수 있음

### 4. 전자도서 및 출판

**(1) 전자도서**
* 전용 뷰어

**(2) 전자출판**
* 웹진
* POD (Publish On Demand)
* EPUB(Electronic Publication)
  * 개방형 자유 전자서적 표준 
  * DRM (Digital Rights Management)

# 4강. 멀티미디어 데이터(2) - 사운드

## 3.2 사운드

### 1. 사운드의 개념


**(1) 아날로그 사운드**
* 사운드의 진폭, 주기, 주파수
  * 진폭 : 파형의 최고점 또는 최저점과 중앙선과의 파장의 높이
  * 주기 : 한 사이클이 발생하는 시간
  * 주파수 : 단위 시간당 사이클의 수(Hz: Hertz)
* 음의 범위
  * 음의 범위는 주파수로 구분
* 아날로그 사운드는 음색, 음의 크기, 음의 높이로 구성
  * 음색 : 주파수와 진폭에 의해 결정
  * 음의 크기 : 진폭 부분이 클수록 사운드의 크기도 큼
  * 음의 높이 : 주파수라고 표기한 사이클이 1초에 반복되는 수

**(2) 디지털 사운드**

### 2. 사운드의 변환

* ADC(Analog-to-Digital Converter) : 아날로그 신호가 샘플링 -> 양자화 -> 부호화 과정을 통해 디지털 신호로 변환

**(1) 샘플링**
* 파형을 일정한 간격으로 잘게 쪼개어 각 지점의 값을 하나하나 차례대로 끄집어내는 작업

> **나이퀴스트 정리(Nyquist theorem)**  
> : 샘플링 이론으로, 원래의 아날로그 신호를 재현하기 위해서는 아날로그 신호가 갖는 최고 주파수의 2배 이상을 샘플링해야 한다.  
> 예로, 우리가 들을 수 있는 소리를 샘플링하기 위해서는 가청주파수의 최대치인 20kHz 이상의 샘플링 주기가 필요. 고품질을 유지하기 위해서는 4배 이상의 샘플링 주기를 사용
> 


**(2) 양자화**
* 양자화는 어떤 값을 취할 수 있는 유한의 이산적인 값으로 표현하는 조작

**(3) 부호화**
* 양자화된 아날로그 신호들을 디지털 형태의 2진 정보로 표현하는 것

> 디지털 사운드를 수치적으로 나타내는 기준
> * 샘플링 크기
> * 샘플링 비율
> * 채널의 수 (모노, 스테레오)

### 3. 사운드 변환 방법

**(1) PCM**
* 입력한 값을 그대로 저장하는 방법
* 아날로그 파형을 작은 시간 간격으로 연속적으로 나누어 각기 직사각형 형태의 크기로 표시한 후 이의 높이를 숫자화함

**(2) DPCM**
* 이전의 값과의 차이만을 저장하는 방식으로, PCM 방식에 비해 데이터 양이 줄어듦

**(3) ADPCM**
* 예측 부호화 방식
* 과거의 음성 신호를 기준으로 다음에 들어올 신호의 크기를 예측하고, 실제의 입력 신호로부터 빼줌으로써 오차 신호를 발생시켜 이 오차 신호를 양자화


### 4. 디지털 오디오의 생성과 저장

**(1) 디지털 오디오의 생성**
* 디지털 오디오 파일의 크기
  * 샘플링 크기와 샘플링 비율에 따라 다름
  * 오디오 파일의 크기 == 샘플링 크기(byte) x 샘플링 비율(Hz) x 채널 개수 x 시간(초)

**(2) 디지털 오디오의 저장**
* WAV
* AU
* AIFF
* RealAudio
* ASF

### 5. 디지털 오디오 편집 소프트웨어
* 사운드 포지
* 골드 웨이브

### 6. 미디

**(1) 미디의 개요**
* Musical Instrument Digital Interface
* 실제 소리를 갖고 있는 것이 아니라 악기의 연주 방법에 대한 정보

**(2) 미디 편집 프로그램**
* 시퀀서
* 노테이션 프로그램
* 보이싱 프로그램

**(3) 미디 파일 형식**
* SMF
* RMI
* MOD

# 5강. 멀티미디어 데이터(3) - 이미지와 그래픽

## 3.3 이미지와 그래픽

### 1. 이미지

**(1) 개요**
* 픽셀
  * 화면을 구성하는 기본 단위, 비트맵 방법
  * 각 픽셀에 할당된 비트의 수가 클수록 더 많은 컬러
* 해상도 (resolution)
  * 이미지의 선명함을 수치로 표현
  * 단위 길이당 표시할 수 있는 픽셀의 수. dpi (dot per inch)
  * 장치 해상도
  * 이미지 해상도

**(2) 이미지의 표현**
* 래스터 방식
  * 픽셀 단위로 표현
* 벡터 방식
  * 기하학적인 객체들로 표현

**(3) 컬러 모델**
* 디지털 환경에서 컬러를 표현하는 방법, 컴퓨터에서 각 컬러를 규정지어 제현할 수 있는 색 공간
* 색의 세 가지 속성인 색상(hue), 명도(lightness), 채도(saturation)를 3차원 공간의 좌표로 나타낸 것

* RGB 모델 : 가산 혼합 방식. 모니터
* CMY 모델 : 감산혼합 모델. CMYK
* 기타 모델
  * 인간의 직관적인 시각에 기초하여 만들어진 컬러 모델
    * HSV(HSB) : 색상(hue), 채도(saturation), 명도(value) / brightness / 원뿔 모양
    * HSL : 원뿔 두 개 이용
    * HSI : Intensity, 영상 처리 분야
  * Y는 휘도(luminance). YIQ와 YUV 모델은 조명을 사용하는 TV 방송에 적합
    * YIQ 
    * YUV
    * YCbCr : JPEG과 같은 압축 방식에 사용

**(4) 이미지 처리(영상 처리)**
* 개념 : 그림을 컴퓨터 기술로 다루는 분야
* 영상 처리 기법
  * 포인트 처리 : 화소의 원래 값이나 위치를 기반으로 화소 값을 변경
    * 영상의 밝기 값 조절 : 그레이 영상(0~255)
    * 히스토그램 : X축은 픽셀의 밝기, Y축은 픽셀의 빈도수 / 히스토그램 평활화
  * 영역 처리
    * 블러링 : 노이즈 등을 제거
    * 에지 추출 : 윤곽을 추출
  * 기하학적 처리
  * 프레임 처리
* 영상 처리의 응용 분야
  * 과학
  * 영화
  * 의료산업
  * 머신 비전
  * 법
  * 기타


### 2. 그래픽

**(1) 컴퓨터 그래픽스**
* 개요
  * 컴퓨터 그래픽스는 시각적이고 기하학적인 정보를 계산 기법을 이용하여 처리하거나 조작하는 과정에 대하여 연구
* 컴퓨터 그래픽스 기법
  * 래스터 그래픽스(2D), 벡터 그래픽스 (3D)
* 컴퓨터 그래픽스 소프트웨어

**(2) 3D 그래픽스**
* **모델링**
  * 모델링이란 물체의 기하학적 형상을 제작하는 것을 말함
  * 3차원 스캔에 의한 모델링
  * 모델링의 대상 물체를 3차원 좌표계를 사용하여 컴퓨터로 물체의 모양을 표현하는 과정
  * 와이어 프레임 모델 : 
    * 점과 선을 기반으로 물체의 골격을 구성하는 모델
  * 다각형 표면 모델 (Polygon Surface Model): 
    * 삼각형이나 사각형 같은 면을 이용하여 물체를 표현
  * 솔리드 모델 : 
    * 면 중심의 모델링 방법을 개선한 것으로서, 물체 내부가 꽉 차 있는 형태로 물체를 구성
* **렌더링**
* 랜더링이란 모델링된 물체를 실제 형태와 같은 영상으로 바꾸는 것
* 3차원으로 모델링된 물체를 사실적으로 보이게 하기 위해 보이지 않은 면을 제거하고, 보이는 면을 명암과 색상, 질감 등을 입혀 원근감을 부여함으로써 실제에 가까운 물체의 영상으로 바꾸는 방법
  * 매핑 기법
    * 물체의 사실감을 높이기 위해 텍스처를 사용하요 질감처리를 하는 기법 
      * 텍스처 매핑
      * 범프 매핑
      * 불투명 매핑
      * 반사 매핑
  * 쉐이딩 기법
    * 물체의 각 단위 면의 밝기를 광원과 주위의 밝기 그리고 입체면의 각도 등을 계산하여 색을 입히는 기법
      * 플랫 쉐이딩
      * 고러드 쉐이딩
      * 퐁 쉐이딩
  * 은선 제거 기법
    * 물체의 보이지 않는 선을 제거하는 기법

### 3. 이미지와 그래픽의 파일 형식

* 래스터 방식
  * BMP, ICO, PCX, GIF, PNG, JPEG, TIFF, WebP
* 벡터 방식
  * CGM, SVG, DXF, AI, CDR
* 혼합 방식
  * PDF, PICT, SWF, EPS, WMF
* 기타 파일

### 4. 이미지와 그래픽의 편집 소프트웨어
 
* 이미지 편집도구
  * 포토숍, 페인트숍 프로, 김프 등
* 2D 그래픽스 편집도구
  * 일러스트레이터, 코릴드로우 등
* 3D 그래픽스 소프트웨어
  * 3ds Max, 마야 등

# 3.4 비디오와 애니메이션

## 1. 비디오

### (1) 비디오의 개요
* 동영상 : 정지 이미지들의 연속적인 집합체
* 프레임 : 비디오를 구성하고 있는 이미지 하나하나
* 비디오는 화면의 크기와 초당 프레임 수로 구분

### (2) 아날로그 비디오와 디지털 비디오0
* 아날로그 비디오
  * 아날로그 비디오의 특성
    * 프레임 속도
    * 주사선의 수
    * 영상비
  * 아날로그 비디오의 연결 형식
    * 컴포짓 
      * 가장 단순한 연결방식
      * 루미넌스 신호와 컬러 신호를 합쳐서 전송하기 때문에 가장 질이 떨어짐
    * S-비디오
      * 콤포짓보다 한 단계 높은 방식
      * 루미넌스 신호는 하나의 선으로 보내고, 두 개의 컬러신호는 다른 선으로 보내는 방식
    * 컴포넌트
      * 가장 우수한 연결 방식
      * 각각의 YCC 신호를 각각의 테이블을 통해 전송
* 디지털 비디오
  * 샘플링 방법과 속도
  * 데이터 속도와 프레임 속도

### (3) 아날로그 및 디지털 편집
* 아날로그 편집
  * 선형 편집
  * 아날로그 편집 장비
* 디지털 편집
  * 비선형 편집
  * 디지털 편집의 장.단점
    * 장점
      * 녹화된 영상의 특정 부분에 대한 랜덤 액세스가 가능
      * 편집시스템의 구성이 간단
      * 화질의 열화가 없는 편집이 가능
      * 다양한 효과와 자막 등의 손쉬운 구현
      * 보존성이 우수
    * 단점
      * 소프트웨어적인 처리 때문에 랜더링 시간이 많이 걸림
      * 각 포맷 간의 호환성 부족
      * 컴퓨터를 이용한 작업이 이루어지므로 전문성 필요
  * 디지털 편집 장비
    * 입력 장비: 카메라
    * 캡쳐보드(코덱보드)
* 비디오 테이프

### (4) 비디오의 압축 코덱
* MPEG-4
* WMV
* X.264
* DivX

### (5) 비디오 파일 형식

### (6) 비디오 편집 소프트웨어
* 윈도우 무비 메이커
* 어도비 프리미어 프로
* 미디어 컴포저

## 2. 애니메이션

### (1) 애니메이션의 개요
